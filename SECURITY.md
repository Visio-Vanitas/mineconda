# Security Policy

## Supported Versions

Security fixes are only guaranteed for the latest code on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for an undisclosed security vulnerability.

Preferred process:

1. Use GitHub private vulnerability reporting if it is enabled for this repository.
2. If private reporting is unavailable, contact the maintainers privately through GitHub before public disclosure.
3. Include affected versions, impact, reproduction details, and any suggested mitigation if available.

We will triage the report, confirm impact, and coordinate a fix and disclosure when appropriate.
